1. Get the application URL by running:
  export POD_NAME=$(kubectl get pods --namespace {{ include "toy-load.namespace" . }} -l "app={{ include "toy-load.name" . }}" -o jsonpath="{.items[0].metadata.name}")
  kubectl port-forward --namespace {{ include "toy-load.namespace" . }} $POD_NAME 9090:{{ .Values.config.port }}
  echo "Visit http://127.0.0.1:9090/work"

2. Metrics are exposed on {{ .Values.config.metricsPath }}. When `prometheusOperator.enabled=true` a ServiceMonitor is installed; otherwise pod annotations enable vanilla Prometheus scraping.

3. If Grafana sidecar scanning is enabled, a dashboard ConfigMap named {{ include "toy-load.fullname" . }}-dashboard is created with label grafana_dashboard=1.

4. HPA is enabled by default (autoscaling/v2). Tune via values.yaml under autoscaling.* and ensure metrics-server is running.
